# Contributing to ToddlerMode

Thank you for your interest in contributing to ToddlerMode! This document provides guidelines and instructions for contributing to this project.

## How to Contribute

### Reporting Bugs

If you find a bug in the application, please create an issue with:

1. A clear, descriptive title
2. A detailed description of the bug
3. Steps to reproduce the issue
4. Expected behavior vs. actual behavior
5. Screenshots if applicable
6. System information (OS version, .NET version)

### Suggesting Features

Have an idea for a new feature? Please create an issue with:

1. A clear, descriptive title
2. A detailed description of the proposed feature
3. Any relevant mockups or diagrams
4. The problem this feature would solve

### Code Contributions

1. Fork the repository
2. Create a new branch for your feature or bugfix
3. Make your changes
4. Test your changes thoroughly
5. Submit a pull request with a clear description of the changes

## Development Setup

1. Install .NET 6.0 SDK or later
2. Clone your fork of the repository
3. Build the project using `dotnet build`
4. Run the project using `dotnet run`

## Coding Standards

- Follow Microsoft's C# Coding Conventions
- Use meaningful variable and function names
- Comment your code where necessary
- Write unit tests for new functionality

## Pull Request Process

1. Update the README.md and documentation with details of changes if needed
2. Update the version number in any relevant files
3. Your pull request will be reviewed by a maintainer
4. Once approved, your pull request will be merged

## Code of Conduct

### Our Pledge

We are committed to providing a friendly, safe, and welcoming environment for all contributors.

### Our Standards

Examples of behavior that contributes to creating a positive environment include:

- Using welcoming and inclusive language
- Being respectful of differing viewpoints and experiences
- Gracefully accepting constructive criticism
- Focusing on what is best for the community

### Enforcement

Instances of abusive, harassing, or otherwise unacceptable behavior may be reported to the project maintainers. All complaints will be reviewed and investigated promptly and fairly.
